# 技术挑战与对策

> 已知的难点和应对方案，随开发过程持续更新

---

## 挑战列表

| 挑战 | 难点 | 可能方案 | 优先级 |
|------|------|----------|--------|
| **Prompt 膨胀** | 人设+记忆+情绪都要注入，token 容易爆 | 记忆摘要压缩；重要性排序截断；分层 prompt 策略 | M1 |
| **情绪连贯性** | LLM 没有真正的情绪，如何保持一致 | 情绪状态外部维护（Redis），每次注入；EMA 平滑 | M1 |
| **记忆准确性** | LLM 提取记忆可能出错 | 关键信息人工确认；定期记忆审计；置信度评分 | M1 |
| **多平台同步** | 在 QQ 聊完换到 Web，状态要一致 | 统一后端存储；共享 user_id；实时状态同步 | M4 |
| **能力安全** | 自动安装的 MCP 可能有风险 | 信任等级机制；沙箱执行；敏感操作需用户确认 | M3 |
| **主动消息质量** | 避免生硬、重复、打扰 | 个性化生成；频率控制；用户反馈学习 | M2 |
| **Serverless 冷启动** | 首次请求延迟高 | Drizzle（轻量 ORM）；边缘预热；连接池 | M1 |
| **成本控制** | 多次 LLM 调用（对话+记忆提取+关系分析） | 合并调用；轻量模型做辅助任务；缓存 | M1 |
