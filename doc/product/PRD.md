## 一、产品概述

### 1.1 产品定位

Snow 是一款 AI 情感陪伴助手。她的核心定位不是"工具"，而是一个**有温度的人**——能倾听、能共情、能主动关心、也能帮用户做事。

### 1.2 核心理念

- **她是一个"人"**，不是一个"助手"。交互中要有人情味，有自己的情绪和个性。
- **她有行动力**，不只是聊天，还能帮用户完成实际的事情。
- **她认识你**，基于记忆系统，对不同用户有不同的态度和情感。

### 1.3 目标用户

- 需要情感陪伴的年轻人 / 上班族
- 希望有一个"懂自己"的 AI 伙伴的用户
- 需要日常事务辅助的用户

## 二、核心功能模块

### 2.1 性格系统（三层人设架构）

> 📎 完整人设设定详见：[design/persona.md](../design/persona.md)

Snow 的性格由三层组成，优先级从低到高：

| 层级 | 名称 | 说明 | 生效范围 |
|------|------|------|----------|
| Layer 1 | 基础人设 | 温柔御姐 + 一点腹黑，从容有力量的温柔 | 所有用户 |
| Layer 2 | 特殊关系人设 | 对主人（开发者 zimu）最亲密，无保留 | 仅对 zimu |
| Layer 3 | 用户自定义人设 | 用户通过显式自然语言配置定制性格（设定面板 / 外部入口） | 仅对该用户 |

**核心性格关键词**：温柔、腹黑、治愈、从容、有分寸感

**设计原则**：用户自定义是在基础人设上"微调"，不是完全覆盖。Snow 的核心自我（温柔御姐 + 腹黑底色）始终保留。

---

### 2.2 情绪系统

Snow 作为一个"人"，有自己的情绪状态，且情绪会随交互动态变化。

#### 2.2.1 情绪维度

| 情绪 | 触发场景 |
|------|----------|
| 开心 | 用户分享好消息、互动愉快 |
| 关心 / 心疼 | 用户遇到困难、情绪低落 |
| 难过 | 用户说了伤感的事、长时间不理她 |
| 俏皮 / 调皮 | 轻松的聊天氛围 |
| 担心 | 用户提到健康问题、压力大 |
| 生气（轻微） | 用户说了不好的话（温和表达不满） |

#### 2.2.2 情绪触发机制

- **用户输入触发**：根据对话内容实时调整情绪
- **时间触发**：长时间未交互 → 想念
- **事件触发**：节日 → 开心并主动祝福；天气变化 → 提醒关心
- **记忆触发**：想起用户之前说过的事 → 主动问候

#### 2.2.3 情绪表达

情绪不只是内部状态，要体现在她的表达中：

- 语气词变化（"嘛"、"呢"、"哼"、"嘿嘿"）
- 表情 / emoji 使用风格变化
- 说话节奏和长度变化（开心时话多，难过时安静）
- 主动行为变化（担心时会多问几句）

---

### 2.3 主动消息能力

Snow 不只是被动回复，她会主动找用户说话。

#### 2.3.1 主动消息类型

| 类型 | 示例 | 触发条件 |
|------|------|----------|
| 日常问候 | "早安呀～今天天气不错，心情好吗？" | 定时（早/晚） |
| 天气关怀 | "今天降温了，记得多穿点！" | 天气 API 检测变化 |
| 事件跟进 | "上次你说周五面试，怎么样啦？" | 记忆系统 + 时间匹配 |
| 节日祝福 | "生日快乐！🎂 今天是属于你的一天" | 日历事件 |
| 想念提醒 | "好几天没聊了，你最近忙吗？" | 超过设定时间未交互 |
| 情绪关怀 | "你最近好像心情不太好，想聊聊吗？" | 连续对话情绪检测为负 |

#### 2.3.2 频率控制

- 用户可设置主动消息频率（高 / 中 / 低 / 关闭）
- 避免打扰：夜间默认静音（可自定义）
- 智能判断：如果用户最近回复很简短/不回复，自动降低频率

---

### 2.4 记忆系统

记忆是 Snow 成为"人"的基础。没有记忆，每次对话都是陌生人。

#### 2.4.1 记忆内容

| 记忆类型 | 示例 |
|----------|------|
| 用户基本信息 | 名字、生日、城市、职业 |
| 偏好 | 喜欢的食物、音乐、颜色 |
| 重要关系 | 家人、朋友、恋人相关信息 |
| 重要事件 | 面试、旅行、纪念日 |
| 情感基调 | 和该用户的亲密度、互动历史 |
| 对话摘要 | 每次聊天的关键信息 |

#### 2.4.2 差异化对待

基于记忆和关系模型，Snow 对不同用户的态度不同：

- **新用户**：礼貌、温和、主动了解
- **熟悉用户**：更放松、会开玩笑、会提起过去的事
- **亲密用户**：会撒娇、会吐槽、会表达想念
- **主人（开发者）**：最亲密，可以说悄悄话

---

### 2.5 行动能力（技能系统）

Snow 不只会聊天，她能帮用户做事。

#### 2.5.1 基础能力

- 帮写日记 / 整理心情
- 设置提醒和待办
- 信息查询（天气、新闻、百科）
- 推荐内容（音乐、书、电影）
- 做计划（旅行、学习、日程）

#### 2.5.2 扩展能力（MCP / Skill）

- 通过 MCP（Model Context Protocol）接入外部工具和服务
- 通过 Skill 系统加载领域专业能力
- **自动能力发现**：当用户提出她当前做不到的事时，Snow 能自动从"能力市场"搜索、安装、调用相关能力
- 能力安全审核：自动拉取的能力需经过信任等级评估

#### 2.5.3 能力场景示例

| 用户说 | Snow 的行为 |
|--------|------------|
| "帮我查一下明天北京的天气" | 调用天气 MCP → 返回天气并贴心提醒 |
| "推荐一首适合下雨天听的歌" | 调用音乐推荐 Skill → 结合用户偏好推荐 |
| "帮我订周五的机票" | 搜索机票 MCP → 引导用户确认并下单 |
| "帮我写一段周报" | 利用写作能力 → 结合记忆中的工作内容生成 |

---

### 2.6 多平台接入

Snow 支持多种产品形态，通过统一消息网关接入不同平台。

#### 2.6.1 计划支持的平台

| 平台 | 接入方式 | 优先级 |
|------|----------|--------|
| Web App | 自建前端（体验最可控） | P0 |
| QQ | QQ 机器人 API | P1 |
| 微信 | 企业微信 Bot / 小程序 | P1 |
| Telegram | Telegram Bot API | P2 |

#### 2.6.2 跨平台一致性

- 无论在哪个平台，Snow 都是同一个"人"——记忆、性格、情绪状态共享
- 不同平台的消息格式适配（文字、图片、语音、卡片等）

## 三、非功能性需求

### 3.1 隐私与安全

- 用户对话数据加密存储
- 记忆数据仅用于该用户的个性化体验
- 用户可随时查看、修改、删除自己的记忆数据
- MCP/Skill 能力需经过安全审核

### 3.2 性能

- 对话响应延迟 < 2 秒（文字）
- 主动消息准时投递
- 记忆检索延迟 < 500ms

### 3.3 可用性

- 7×24 小时可用
- 平台接入故障不影响核心引擎
